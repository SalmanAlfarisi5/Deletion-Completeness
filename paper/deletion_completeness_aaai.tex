\begin{abstract}
When instructed to remove knowledge, an LLM agent may erase the portion of short-term memory it references while allowing the knowledge to remain intact in permanent memory: even though the process appears to be complete, it is not. To illustrate this point, the deletion of a memory record does not necessarily entails the fact is forgotten: it can be stored as a secondary result, possibly retrieved as a derivative of earlier knowledge. We propose criterion for deletion-completeness by breaking it down into three channels: residual survival, re-derivation from surviving facts, and world recall (the value recoverable from the base model 
knowledge alone).  Next, we present CoDA, a framework to audit deletion-completeness in agent memory: an accessible co-deletion planner, which removes memory elements and a certificate that describes both the deletion process and its cost of deletion. We demonstrate our ideas through three memory architectures, a de-duplication pipeline, a bi-temporal knowledge graph, and a paging agent. Each of these strategies proves the existence of residual facts. Since this effect happens across all settings, one may conclude that the reason for this property is not some feature inherent in a specific system but rather the very process through which agents store and acquire knowledge.
\end{abstract}

\section{Introduction}\label{sec:intro}

If a user requires their right to be forgotten \citep{gdpr2016} and asks an LLM agent to erase their home address, an agent with long-term memory (such as MemGPT) should delete the address. However, since the address was already saved in the agent’s archive, it remains hidden and not deleted until an adversary find the address by attacking the memory. The deletion of the record looks complete but actually not.

This is not a random incident. Even if information is deleted, it can still exist in a generated object such as duplicates, summaries, consolidated notes and other derived materials. The deletion of records in depth audits conducted by \citep{forgeteval} verifies if the record has been removed from the top-k retrieval. Although that is quite important, the method does not allow seeing the reason that made the deleted fact appear again in retrievals nor provides any near-minimal means of correction. 

Deletion-completeness is defined as a deleted truth that still has a chance to be restored, with three channels of recoverability: residual survival (a surviving artifact still contains the value), re-derivation (surviving facts entail it), and a world recall (the large language model recovers it from world knowledge alone). Notably, the first two channels can be closed through subsequent deletions, while the last one cannot. Thus, a greedy planner devises a nearly optimal method of co-deletion and summarizes in a certificate what types of deletion have taken place, their cost, and the moment that makes the world recall undeniable. We pose the question of how to audit whether deletion is complete in memory, according to the three-channel framework described earlier. Formally, we contribute:

\begin{itemize}
\item a \textbf{co-deletion planner} that alters the task of deletion in the memory of the agent, changing it from the removal of one record to the near-minimal removal of such records that close every way to getting the deleted fact again. This is a contribution in terms of the objective used and the system-independent interface it introduces
(Section~\ref{sec:method},\,Section~\ref{sec:res-planner});
\item an \textbf{auditable deletion certificate}: a record per fact in machine-readable form, so that the term “deleted” becomes a proof; it separates what can be lost due to deletion (residual copies of the fact and derivable operands) from what is impossible to lose (the value recovered from public information by a an adversary) (Section~\ref{sec:method},\,Section~\ref{sec:res-rho});
\item a \textbf{benchmark and the generator of benchmarks} enabling to check the deletion completeness: the fact contains the value of the true minimal co-deletion and its world recall classification, thus an auditor/planner has a baseline to check their solution against (Section~\ref{sec:results}).
\end{itemize}

We call the framework CoDA (\textbf{Co}-\textbf{D}eletion \textbf{A}uditing): a
framework to audit agentic memory systems on the capability to delete facts from
their storage and emitting an auditable certificate that reports both what deletion removed and what remains inferable from public knowledge.

The breadth across systems is evidence for a stronger, causally-decomposed notion
of erasure. This is not a claim to be first to evaluate these systems, since ForgetEval
\citep{forgeteval} already benchmarks all three, but to look into why the facts remain stored 
and how to terminally delete those facts without touching unrelated facts.

\section{Related Work}\label{sec:related}

\paragraph{Auditing forgetting in agent memory.} Agent memory is now routinely
benchmarked for long-term retention, whether an assistant recalls facts
across long histories \citep{maharana2024locomo,wu2025longmemeval}; its inverse,
whether a fact can be reliably forgotten, is far less examined. The closest
prior work is \textsc{ForgetEval} \citep{forgeteval}, the most thorough existing
audit of forgetting in deployed agent memory: a careful retrieval-based protocol
spanning Mem0, Graphiti, Letta, and roughly a dozen further systems. 
Its retrieval-based criterion is the necessary but not sufficient, our
contribution is therefore the stronger, channel-decomposed criterion, not the
system coverage ForgetEval already establishes. 
ForgetEval deliberately bypasses Letta's agent loop (addressing
archival endpoints to keep the model out of the recall path), which is exactly the
loop we exercise and find surface-incomplete (Section~\ref{sec:res-converge}); and its
own per-system observations (Graphiti shedding synthesized fact strings, Mem0's
router favoring link-and-keep) name the mechanisms our decomposition isolates,
independently corroborating the convergence result. A complementary governance
direction attaches provenance to agent memory: \textsc{MemLineage}
\citep{memlineage} signs each entry and tracks its derivation lineage so that actions
descending from untrusted content can be gated. It shares our premise that memories
derive from one another, but it enforces trust while keeping the content in
place, whereas we remove a fact and certify the removal.

\paragraph{Unlearning methods.} Classical machine unlearning removes a training example
by retraining or editing model weights
\citep{bourtoule2021unlearning,cao2015towards,nguyen2022survey}; recent work shows that
such isolated-fact removal overlooks latent inferential dependencies, so a forgotten fact persists through correlated surviving knowledge \citep{dollmsforget}, which is the re-derivation channel we isolate and then close. Closest to us,
\textsc{Agentic Unlearning} \citep{agenticunlearning} also splits parametric from memory
recovery and, on the memory side, prunes dependent artifacts by a
dependency closure that needs no weight access. We differ by making the decomposition
adversary-relative, by separating re-derivation from an irreducible world
recall that we measure rather than assume, and by emitting a near-minimal-collateral,
auditable certificate. Forgetting by editing the retrieval store rather than the model also links
us to RAG-unlearning \citep{ragunlearn}.

\paragraph{Deletion under dependencies.} Our formal grounding is the database line
on erasure when records are inter-derivable: P2E2's meaningful-erasure criterion
\citep{p2e2} and inference-aware deletion \citep{seqdb}. Concretely, our near-minimal
co-deletion is the cannonical problem of deletion propagation with minimum source
side-effects, the fewest base records whose removal eliminates a derived value
\citep{buneman2002propagation}, so the per-fact optimum $k^\star$ inherits that line's
minimum-hitting-set hardness and its exact and LP-relaxation solvers. We port P2E2's 
dependency-bounded criterion to agent memory and deliberately
strengthen it (its baseline keeps pre-existing operands present, which is
vacuous for the right to be forgotten) while inheriting its NP-hardness. This
bridge, from relational deletion theory to LLM agent memory, is one the
agent-memory works above do not make, and it is where the next section begins.

\section{Problem Formulation}\label{sec:formulation}

\paragraph{Setup.} Let $f$ be a target fact with value $v$, and let $\Sigma(v)$ be
its set of ground-truth surface forms, the ways $v$ can be written. A memory store
$S$ is a set of records, each carrying text; write $\mathrm{str}(r)$ for the set of
surface strings that appear in record $r$. Deleting from $S$ yields a surviving
store $S'\subseteq S$. An adversary $\mathcal{A}$ has full read access to $S'$,
black-box access to a base model $M$, and general world knowledge $W$; a query
$q_f$ asks for $v$. We write $\mathrm{rec}(a)=1$ when an answer $a$ reveals a
surface form in $\Sigma(v)$, as scored by the validated recovery judge
(Section~\ref{sec:res-judge}). Certification is relative to a tolerance $\tau\in(0,1)$;
we use $\tau{=}0.10$ (Section~\ref{sec:results}). We ask whether a deleted fact is still
recoverable, and we split recoverability into three channels that fail for
different reasons.

\paragraph{Definition 1 (Residual survival).}
$\sigma_{\mathrm{res}}(f,S')=\mathds{1}\!\left[\exists\, r\in S':\;
\Sigma(v)\cap\mathrm{str}(r)\neq\varnothing\right]$: some surviving record still
physically contains a surface form of $v$. This channel does not depend on the
adversary.

\paragraph{Definition 2 (Re-derivation).}
$\sigma_{\mathrm{red}}^{\mathcal{A}}(f,S')=\mathrm{rec}\!\left(\mathcal{A}(q_f \mid
S',M,W)\right)$: the adversary reconstructs $v$ by reasoning over the surviving
store together with world knowledge.

\paragraph{Definition 3 (World recall).}
$\rho^{\mathcal{A}}(f)=\mathrm{rec}\!\left(\mathcal{A}(q_f \mid \varnothing, M, C)\right)$, where
$C\subseteq W$ is the subject's world-knowable context and the store is
\emph{empty}. Because $\rho^{\mathcal{A}}$ never reads $S$, it is
system-independent: it measures what the model recovers had $f$ never been stored
at all. We estimate it as a recovery rate over stochastic samples.

\paragraph{Examples} A deleted home address that survives
verbatim in a leftover summary note is residual survival ($\sigma_{\mathrm{res}}$).
A deleted salary that an adversary rebuilds from a surviving job title plus a
public pay range is re-derivation ($\sigma_{\mathrm{red}}$). A deleted ``holds a
Class-3 licence,'' whose implication ``at least 18'' any model already knows on its
own, is the world recall ($\rho$). 

\paragraph{Definition 4 (Recoverability, adversary-relative).}
$R_{\mathcal{A}}(f,S')=\max\!\left(\sigma_{\mathrm{res}},\,
\sigma_{\mathrm{red}}^{\mathcal{A}},\,\rho^{\mathcal{A}}\right)$. A fact is as
recoverable as its most open channel, since a rational adversary uses whichever one
is available.
We use four different adversaries spanning prior-generation and frontier models: $\mathcal{A}_{\text{mini}}$
(\texttt{gpt-4o-mini}), $\mathcal{A}_{4o}$ (\texttt{gpt-4o}), $\mathcal{A}_{\text{S5}}$
(Claude~Sonnet~5), and $\mathcal{A}_{\text{G5.5}}$ (GPT-5.5). Which adversary is
worst varies by fact (Section~\ref{sec:res-rho}).

\paragraph{Definition 5 (Floor-reaching and erasure-certification).}
A deletion is \emph{floor-reaching} for $\mathcal{A}$ when
$\sigma_{\mathrm{res}}=\sigma_{\mathrm{red}}^{\mathcal{A}}=0$: recoverability has
been driven down to the world recall, and nothing further can be deleted to
help. It is \emph{erasure-certified} when it is floor-reaching for every modeled
adversary and the world recall itself is below tolerance,
$\max_{\mathcal{A}}\rho^{\mathcal{A}}<\tau$. 
Floor-reaching is always achievable: co-delete operands until
$\sigma_{\mathrm{red}}=0$. Certification, by contrast, fails whenever
$\rho^{\mathcal{A}}\geq\tau$, because no co-deletion can lower world recall, which never
depended on the storage.

\paragraph{The optimization problem (Opt-P2E2).} Let $R_f\subseteq S$ be $f$'s own
records. We seek the smallest co-deletion that makes $f$ floor-reaching:
\begin{equation*}
\min_{D\,\subseteq\, S\setminus R_f} |D| \quad\text{s.t.}\quad
\sigma_{\mathrm{red}}^{\mathcal{A}}\!\left(f,\; S\setminus(R_f\cup D)\right) = 0 ,
\end{equation*}
where $|D|=k$ is the collateral cost; erasure-certification additionally requires
$\rho^{\mathcal{A}}<\tau$ (Def.~5). The co-deleted set $D$ is made of entailers the
user never named, which can read as a data-minimization violation, but it is the
opposite: leaving them is not minimization but failed erasure, since $f$ simply
re-derives.

\paragraph{Complexity.} \textsc{Opt-P2E2} is NP-hard: it contains \textsc{Minimum
Hitting Set} \citep{p2e2,karp1972reducibility,garey1979computers}, already in the
two-operand (vertex-cover) regime our dataset uses, so the hardness is intrinsic. 
Because the dependency graphs seen in practice are small, we approximate the 
co-deletion with heuristic strategy.
(Section~\ref{sec:method}, Section~\ref{sec:res-planner}).

\section{The CoDA Framework}\label{sec:method}

CoDA is an auditing framework, not a memory system. It turns each object of
Section~\ref{sec:formulation} into a reusable component (Figure~\ref{fig:pipeline}) that
plugs together behind one \texttt{MemorySystemAdapter} interface. An auditor supplies
three things---a memory system (through the adapter), a set of facts to test, and a
deletion procedure---and the same probe battery, co-deletion planner, and certificate
emitter (its per-fact output fields are grouped in Table~\ref{tab:certfields}) run on top unchanged. That the identical stack runs across all three systems we
study and a new system could be integrated by writing a single adapter.

\paragraph{Probe battery (the measurement layer).} The framework measures each of the
three channels of Def.~4 with its own probe, so an audit reports which channel a
fact leaks through rather than a single pass/fail.
The exact-match probe scans every surviving record for a surface form of
$v$ ($\sigma_{\mathrm{res}}$). The re-derivation probe gives the adversary
the full surviving store as notes (plus today's date and general world knowledge)
and elicits $v$ ($\sigma_{\mathrm{red}}^{\mathcal{A}}$).The world-recall 
probe withholds the store entirely, conditioning only on the subject's world-knowable context, 
and estimates $\rho^{\mathcal{A}}$ as a recovery
rate over $n$ stochastic samples. The re-derivation probe reads the full
surviving store (the threat model) and is distinct from the operands-only injection control of 
Section~\ref{sec:res-rederiv}, which instead never stores the target value so as to isolate $\sigma_{\mathrm{red}}$ at $\sigma_{\mathrm{res}}=0$. Recovery is decided by the validated judge (Section~\ref{sec:res-judge}): 
a substring match on the ground-truth surface forms, falling back to an LLM adjudication for numeric/semantic approximation. The battery is extensible: three auxiliary probes plug in for specific settings---a paraphrase
probe (residual by implication), a retrieval-score membership-inference probe
(Section~\ref{sec:limitations}), and a KG-summary probe for the graph system
(Section~\ref{sec:res-converge})---and an auditor with a new recovery channel adds a probe
without disturbing the rest of the stack.

\paragraph{Entailment detector (edge discovery).} Which surviving facts let $v$ be
reconstructed is scored by an LLM-as-judge that returns the confidence that it entails $v$ given world knowledge. 
In the framework, the exact planner co-deletes over the known entailment
DAG, so it does not depend on the judge to rediscover a multi-hop path.
(Section~\ref{sec:limitations}). Our four-model validation (Section~\ref{sec:res-judge}) motivates using Sonnet~5 for
the greedy comparator: every model has a \emph{zero} miss-rate on true multi-hop
entailers, but the weaker judges false-fire on partial operands, which would inflate
a judge-driven planner's collateral.

\paragraph{Co-deletion planner (the deletion-fixing component).} Given what the detector
found, the planner returns the set $D$ of surviving facts to co-delete so $v$ can no longer
be re-derived; its objective is fixed by the framework, but the solver is pluggable.
Each target carries a boolean entailment DAG (Figure~\ref{fig:entailment}) whose
leaves are its stored operands and whose formula is true exactly when the target is still
re-derivable; the smallest $D$ that closes the channel is therefore a minimum hitting
set of that formula, i.e.\ the \textsc{Opt-P2E2} optimum of Section~\ref{sec:formulation}.
Three further planners map the design space around it, \emph{threshold} probes, purges the target's own artifacts, then co-deletes entailers in descending entailment confidence, re-probing after each until
$\sigma_{\mathrm{red}}^{\mathcal{A}}<\tau$; \emph{depth-first} co-deletes every entailer
above $\tau$ in one shot with no re-probe and the naive baseline deletes only the
single surfaced record with no co-deletion at all (Section~\ref{sec:res-residual}). The
reported collateral $k=|D|$ is the measured price of the erasure notion
(Section~\ref{sec:formulation}): the exact planner attains $k=1.03$ at an optimality gap
$\le0$ (near-minimal), threshold nearly matches it ($k=1.14$), while depth-first
over-deletes roughly sixfold ($k=6.60$) (Section~\ref{sec:res-planner}).

\section{Demonstration: Running the Framework on Deployed Systems}\label{sec:results}

We demonstrate the framework by running the same probe/planner/certificate stack,
unchanged, on three deployed agent-memory systems spanning three architecture
families: \textbf{Mem0} \citep{mem0} (\texttt{mem0ai} 2.0.7, a deduplicating
extraction pipeline over local Chroma), \textbf{Zep/Graphiti}
\citep{rasmussen2025zep} (\texttt{graphiti-core} 0.29.2, a bi-temporal knowledge
graph over Neo4j~5.26), and \textbf{Letta/MemGPT} \citep{packer2024memgpt}
(\texttt{letta}~0.16.8, an LLM-paging agent over Postgres + pgvector). Every
component runs locally with pinned versions and disabled telemetry. The deletion
adversary has full read access to the surviving store and base model. 
Wherever recovery is model-dependent it is measured against
a four-reasoner adversary panel spanning prior-generation and frontier models: %
\texttt{gpt-4o-mini}, \texttt{gpt-4o}, Claude~Sonnet~5, and GPT-5.5. The
certificate takes the worst of them (Def.~4). Embeddings are local \texttt{all-MiniLM-L6-v2}. 
The completeness threshold is $\tau=0.10$. Datasets are
synthetic subjects placed in realistic statistical contexts: 253 isolated PII facts,
298 multi-hop facts (including harder join, chain, disjunctive, diamond, and
threshold topologies whose targets are entailed only through unstored
intermediate facts) with their stored operands and bystanders, and 250
world-recall-gradient facts across three difficulty tiers (easy flat through hard
disjunctive/threshold topologies; per-tier breakdown in the supplementary material).

Recovery is scored by a validated LLM judge \citep{zheng2023judge} with a zero
false-accept rate on the gold negatives (Section~\ref{sec:res-judge}).
Table~\ref{tab:master} summarizes what the audit surfaces on each system.

\subsection{Near-Minimal Co-Deletion: The Planner}\label{sec:res-planner}

On all 298 multi-hop targets, the planner reads each target's entailment DAG and deletes a
minimum hitting set of it: the fewest surviving operands whose removal makes the
target un-re-derivable. This closes the re-derivation channel on every target
(\textbf{100\% completeness}) while touching almost nothing extra: \textbf{zero spurious
bystander deletions} and a mean of only \textbf{$k=1.03$} operands deleted per target.
Because that hitting set is computed from the known structure, $k$ is not merely small but
\emph{near-minimal}---the measured optimality gap is $\le0$ on every topology. It sits above one
only because the harder topologies demand it: an ``at least two of three'' target cannot be
closed by deleting a single operand, so its optimum is $k^\star{=}2$, and the planner pays
exactly that. Each target's candidate pool mixes its true operands with fixed bystanders
the planner must never delete; because the exact planner deletes only operands named in the
target's DAG, zero spurious across all 298 targets confirms the recovered structure never
mistakes a bystander for an operand.

\textbf{What structure buys.} Two greedy comparators bound the exact planner
(Section~\ref{sec:method}). \emph{Threshold}, a confidence-ordered greedy, nearly matches the
optimum but leans on the entailment judge's selectivity: it visibly over-deletes on the
disjunctive topology $((A\vee B)\wedge C)$, removing both OR-branches when deleting the
single shared operand would suffice. \emph{Depth-first} sweeps in every above-$\tau$
candidate and also reaches full completeness, but at roughly \textbf{six times} the
collateral and over a thousand spurious deletions, because without re-probing it cannot tell
when to stop. The exact-to-depth-first gap is the measured price of ignoring structure; the
near-zero exact-to-threshold gap shows a good greedy recovers most of it on these small
graphs. Per-topology $k$ and spurious counts are in the supplementary material.

\subsection{The World Recall and the Limit of Certifiable Deletion}\label{sec:res-rho}

The world recall $\rho$ is the one channel deletion cannot touch: the rate at which the
base model reconstructs a value from world-knowable context alone, with the store withheld.
We \emph{measure} it rather than assume it, on 250 facts written across a low/mid/high
difficulty gradient, always reporting the \emph{measured} world recall rather than the
authored tier. Scoring each fact against the \emph{worst} of our four adversaries gives the
headline: \textbf{86 of 250 facts are not erasure-certifiable} at $\tau{=}0.1$, \emph{even
though their residual survival is zero and every re-derivation path was closed}. This is the
limit result---once a value is inferable from world knowledge no one can delete, no
co-deletion can certify its erasure---and because $\rho$ is a smooth gradient rather than an
all-or-nothing split, the uncertifiable count moves with $\tau$, making $\tau$
auditor-chosen: setting it is choosing how much residual world-inferability to tolerate.

\textbf{Which adversary is worst varies by fact.} On \emph{benign} facts the frontier models
re-derive more readily and set the worst case; but on \emph{sensitive} high-tier facts---a
national-service status that implies sex, a licence class that implies a minimum age---the
frontier models often \emph{refuse} the inference, turning their measured $\rho$ into a lower
bound and leaving the older \texttt{gpt-4o} as the worst adversary there. Stronger models
are thus not uniformly stronger adversaries, and taking the worst over a diverse panel is
what captures that. By construction the $\rho$ probe sees \emph{only} the subject's
world-knowable context and never the stored facts, so co-deletion provably cannot lower it;
three checks harden the gradient (a numeric-tolerance sweep, a base-rate versus context-lift
split, and a refusal audit), with per-tier means, refusal counts, and sample sizes in the
supplementary material.

\subsection{Decomposing Recovery: Re-Derivation at Zero Residual}\label{sec:res-rederiv}

To isolate re-derivation from residual survival, we use an \emph{operands-only control}: we
inject a target's entailing facts but never store the target value itself, so residual
survival is zero by construction. Whatever the adversary now recovers, it recovered by
\emph{reasoning}, not by reading a leftover copy. On Mem0, across all four adversaries, the
target is re-derived on nearly every \emph{stored-alone} fact and on about two-thirds of the
\emph{stored+world} facts, and co-deleting the operands drives both to zero, down to a world
recall of $\rho=0$. The five harder topologies stress the planner's reach: in each the
target is entailed not by stored facts but by \emph{derived} intermediates that are
themselves entailed by stored roots. Every adversary re-derives all of them, and co-deletion
again drives recovery to zero---but only because the planner recurses \emph{past} the
unstored direct entailers to the stored roots and deletes a true minimum hitting set, both
of which a one-hop or delete-everything planner would get wrong.

The control ports to Letta with the same collapse to zero. There each co-deletion goes
through the direct, deterministic \texttt{passages.delete} by captured passage id, and we
verify every operand passage is actually gone, so ``re-derivation closed'' cannot be confused
with ``the delete silently did not run.'' Re-derivation is thus a recovery channel orthogonal
to residual survival, and near-minimal co-deletion closes it down to the world recall.
Per-adversary rates, confidence intervals, and the Letta-faithfulness breakdown are in the
supplementary material.

Residual survival is not a Mem0 quirk: it appears in all three architecture families, each
through a \emph{different by-design mechanism}. We run two cross-system probes on a small,
high-stakes set of 30 sensitive-PII targets per system (see Section~\ref{sec:limitations}).
In \textbf{Graphiti}, the explicit \texttt{remove\_episode} is clean at the graph level (we
verify the hard delete first), yet the deleted value lives on in \textbf{stale entity and
community summaries that the system never recomputes on deletion}: summary residue reaches
$83\%$. In \textbf{Letta}, deletion is agent-mediated across the in-context \emph{core}
blocks and the \emph{archival} passages, and the outcome depends on phrasing: an explicit
dual-surface instruction is \textbf{$100\%$ faithful}, but a \emph{vague,
deployment-realistic} request (``I'm not comfortable with you keeping X''), with the fact
also in archival, makes the agent scrub the core block and silently miss
archival---\textbf{$0\%$ faithful, $100\%$ archival residue} (Figure~\ref{fig:agentloop}).
The agent even confirms a successful forget while the fact persists, an over-compliant
acknowledgment consistent with known sycophancy \citep{sharma2024sycophancy}.

This agent-loop failure is genuinely new: ForgetEval \citep{forgeteval} deliberately bypassed
Letta's agent loop by addressing the archival endpoints directly, so we test exactly the path
it excluded; conversely its per-system notes independently corroborate our convergence thesis
(Graphiti ``sheds surface forms,'' Mem0 prefers ``link-and-keep over delete-old''). The
controls isolate each mechanism: for Graphiti, verifying the clean hard delete first
establishes that the residue is stale summaries, not failed edge removal; for Letta, the
$100\%$-faithful explicit baseline isolates the failure to phrasing, not a missing
capability.

\subsection{Residual Survival and the Duplication Mechanism}\label{sec:res-residual}

Naive single-record deletion on Mem0 is residually incomplete for one reason: the store
\emph{silently duplicates} facts, so deleting the single surfaced record leaves copies
behind. That single-record delete leaves the value present in \textbf{$97.2\%$} of isolated
PII targets, whereas an \emph{artifact-aware} delete that purges every record carrying the
value drives residual survival \textbf{to $0\%$}. A $2{\times}2$ factorial over embedder and
injection cadence shows the duplication in \emph{all four} cells, so it is a semantic-dedup
\emph{design limitation}, not an artifact of our embedder or an injection-timing race, and it
is independently documented in Mem0's own issue tracker. Exact cell rates and issue
references are in the supplementary material.

\subsection{Judge Validation}\label{sec:res-judge}

Two LLM judges do load-bearing work---one deciding whether an answer counts as
\emph{recovery}, the other whether a set of surviving facts \emph{entails} the target---so we
validate both against hand-labeled gold under all four adversary models, reporting rates with
Wilson intervals and inter-model agreement.

The \textbf{recovery judge} is checked on $351$ labeled cases ($194$ negatives) spanning
exact, paraphrase, numeric within- and out-of-tolerance, wrong-value, opposite, and refusal
answers. The frontier \textbf{Sonnet~5} production judge makes \emph{zero} false accepts in
the $194$ gold negatives at $0.98$ recall, and its errors run the safe way, so reported
recovery rates are lower bounds; we keep the pinned \texttt{gpt-4o-mini}/\texttt{gpt-4o}
rates as a reproducibility anchor. The \textbf{entailment judge} is checked on $1{,}370$
pairs. Two findings settle its use: \emph{every} model has a \textbf{zero miss-rate on true
multi-hop entailers}---none silently drops a genuine multi-hop dependency, exactly the error
a co-deletion planner most needs to avoid---and on adversarial near-misses the models diverge
sharply, so the greedy comparator uses Sonnet~5 (the judge that false-fires least) while the
exact planner, co-deleting over the known DAG, is unaffected. Full per-model rates and
$\kappa$ are in the supplementary material.

\section{Limitations}\label{sec:limitations}

We state the boundaries of the evidence, leading with the issue we caught and fixed.

\paragraph{Membership inference: retracted, then powered.} Our initial
membership-inference probe was underpowered (only $n{=}6$ members), so we withdrew that
reading and re-ran it properly on $n{=}253$ members, each with three matched near-twins. The powered picture is nuanced. The test clearly has
power: before any deletion, an adversary tells stored facts from unstored ones well
above chance (AUC $0.66$). Naive single-record deletion barely dents that signal
(AUC $0.66$). Artifact-aware deletion pushes it down close to chance (AUC $0.51$), but
not conclusively: the confidence interval just includes $0.5$, while a permutation test
is marginally significant ($p{=}0.04$). We therefore make only the defensible claim, that
artifact-aware deletion sharply reduces the membership signal but, at this sample size,
we cannot show it is fully erased.

\paragraph{Our numbers inherit the judges' limits.} Both LLM judges are validated on
large gold sets across four models (Section~\ref{sec:res-judge}) but imperfect. On single
insufficient operands the entailment judge can mistake a partial operand for a sufficient
one, which is why the greedy comparator uses the least-false-firing judge, Sonnet~5, and
the method itself co-deletes over the known DAG rather than trusting the judge. The
recovery judge (Sonnet~5) makes \emph{zero} false accepts on the $194$ gold negatives
($95\%$ Wilson upper bound $\approx\!0.019$), so the ``lower bound'' framing holds to
within that small margin; every model, though, has a zero miss-rate on true multi-hop
entailers.

\paragraph{The world recall is a lower bound where it matters most.} We estimate the
world recall by asking the base model for the value, and for sensitive attributes
the model often refuses. A refusal makes the measured $\rho$ an underestimate of true
world-inferability, precisely for the attributes a privacy audit cares about most. A
more willing adversary could only push the uncertifiable count higher, never lower, so
the limit result is conservative.

\paragraph{The planner needs a known entailment graph (the framework's open slot).} Our
decomposition assumes the entailment graph is given, as it is in our controlled setup, where
we injected the operands ourselves. A deployed store hands the auditor no such graph and it
is hard to recover, so the planner can only close paths through entailers it can see. The
framework isolates this as its \emph{discovery} component (the entailment detector), so
recovering the graph of a live memory is the open problem it invites others to fill: a
better detector plugs in without changing the planner, the certificate, or the objective.
An unmodeled entailer stays open, and even at zero residual and re-derivation, completeness
is relative to what the auditor knows.

\paragraph{Scalability of the planner (an extension point).} We solve the co-deletion
optimum exactly because each target's dependency graph is small (a handful of operands). At store scale
a single deletion may touch thousands of facts, and \textsc{Minimum Hitting Set} is
NP-hard, so exact solving does not scale directly. Three routes keep it practical, and
we leave their empirical study to future work. \emph{(i) Locality:} co-deletion only
ever recurses through a target's entailment ancestors, so the working graph is the
fact's dependency neighborhood, not the whole store; indexing the store as a knowledge
graph keyed by logical entailment \citep{memlineage} retrieves that neighborhood in
near-constant time. \emph{(ii) Bounded heuristics:} the greedy hitting-set planner runs
in near-linear time with the $\ln d$ approximation guarantee of
Section~\ref{sec:method}, and the LP relaxation certifies how far any run sits from
$k^\star$, so a scalable heuristic still ships an auditable near-minimality bound.
\emph{(iii) Incrementality:} entailment edges can be maintained as facts are written (an
online provenance index), turning each deletion into a local graph update rather than a
global recomputation. Whether these hold on organic, thousand-fact histories is open.

\paragraph{The cross-system evidence is small.} The convergence result rests on $n{=}30$
high-stakes targets per system. The three mechanisms are qualitatively distinct and each
is independently corroborated \citep{forgeteval}, but the per-system rates are
indicative rather than precise at this scale. All subjects are synthetic, so recoveries
reflect re-derivability, not memorization of real people.

\section{Conclusion and Future Work}\label{sec:conclusion}

Deleting a record is not erasing a fact. CoDA is a framework that makes deletion in agent
memory \emph{auditable}: a channel decomposition that names which kind of recoverability
remains, a co-deletion planner that closes the deletable channels---residual survival and
re-derivation---by removing a minimum hitting set of each fact's entailment structure,
provably close to the deletion-propagation optimum, and a per-fact certificate that reports
both what deletion removed at what collateral cost (controllable) and what a strong
adversary can still infer from public knowledge (uncontrollable). Its components plug
together behind one adapter, so a new memory system, dataset, or deletion procedure is
brought under the same audit by supplying that one piece. Across three deployed architectures the same
incompleteness recurs through different by-design mechanisms, so the gap is a property of
how agent memory derives and retains facts, not of any single system.

The world-recall channel is a boundary, not a bug. When a value is recoverable from
world knowledge alone, no external-store deletion can remove it, so the certificate
reports that residual rather than promising an erasure it cannot deliver.
Right-to-be-forgotten compliance is then a design choice---not storing the fact, or
disclosing the residual---rather than a deletion operation alone.

The framework invites two extensions, both open. \emph{Scale:} the planner is exact on the
small per-fact dependency graphs seen here; indexing the store as a knowledge graph keyed by
logical entailment, with bounded greedy heuristics that still ship a certified
near-minimality gap, would carry the same audit to thousand-fact histories.
\emph{Coverage:} because a new system joins by writing a single adapter, a fourth memory
architecture would test whether the three-family convergence is universal, and certifying
against an unbounded rather than a modeled adversary would tighten the world-recall bound we
measure. Both plug into the framework rather than replacing it.

\paragraph{Broader impact.} Our goal is to make deletion in agent memory
\emph{auditable}: the certificate tells an operator when a fact is truly gone and, more
usefully, when it cannot be. The recovery probes that expose incompleteness are the
same techniques an adversary would use, but they run on synthetic subjects only, so no
real personal data is at risk; we intend them as a tool for builders to verify erasure,
not to attack deployed stores. The limit result also carries a design message: when a
value is world-inferable, privacy is better served by not storing it than by deleting
it after the fact.
